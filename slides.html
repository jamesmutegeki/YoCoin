Blockchain Integration
Custom Python blockchain with Proof-of-Work consensus and MySQL database persistence

Blockchain Architecture

  • Custom Python blockchain implementation with Block and Chain classes
  • SHA-256 hashing for block integrity verification
  • Proof-of-Work consensus algorithm with adjustable difficulty
  • Each block stores timestamp, transactions, proof, previous hash
  • Chain validity verified through hash chain traversal
  • Genesis block created automatically on first startup

DB Persistence

  • Blockchain state persisted in blockchain_blocks and blockchain_transactions MySQL tables
  • Full chain reconstructed from DB on application startup
  • New blocks and transactions atomically committed to both chain and database
  • All loan transactions recorded as immutable blockchain entries

Transaction Lifecycle

Every loan disbursement, repayment, and transfer creates a new transaction that gets mined into the next block.

Immutability

Once a block is added to the chain, it cannot be altered without re-mining all subsequent blocks — ensuring complete transaction integrity.

Transparency

All transactions are verifiable on the chain. Users can view their complete transaction history with block confirmations.

Chain Validation

Built-in verify_chain() method validates block hashes and proof-of-work across the entire chain on every operation.

Security & KYC
Enterprise-grade security with rigorous identity verification for regulatory compliance

Security Features

  • Password Security: bcrypt hashing with strength validation (uppercase, lowercase, digits, special chars, 8+ length)
  • Rate Limiting: Flask-Limiter on login, registration, password reset endpoints
  • Account Lockout: Auto-lock after 5 failed login attempts with timers
  • Two-Factor Auth: TOTP-based 2FA using authenticator apps (Google Authenticator, Authy)
  • Session Security: HTTP-only cookies, SameSite=Lax, 24-hour expiry, secure flag in production
  • CSRF Protection: WTForms CSRF tokens on all forms
  • Input Sanitization: SQL injection prevention, XSS filtering on all inputs
  • Suspicious Activity: Real-time monitoring and alerting system

KYC Process

  • Document Upload: National ID photo + selfie image upload
  • Format Support: PNG, JPG, JPEG, PDF (2MB limit)
  • Magic Byte Validation: File type verification beyond extension checking
  • Admin Verification Queue: Dedicated queue for admin KYC review
  • Status Tracking: Users can check KYC status: Pending / Verified / Rejected
  • Verification Limits: Max 5 upload attempts before manual review required
  • Auto-Reminders: Email/SMS reminders for incomplete KYC submissions
24-hour verification target for KYC documents. Users cannot apply for loans until KYC is verified.

Loan Management
Complete loan lifecycle from application through approval to repayment

Loan Application Process

  • User selects loan amount and purpose (Business, Education, Agriculture, Medical, Personal, Asset Finance)
  • System auto-calculates interest based on credit score tier and amount
  • Repayment schedule generated on approval with due dates and installment amounts
  • Loan calculator available to preview costs before applying
  • Multiple loans allowed only if existing loans are in good standing

Loan States

  • Pending — Awaiting admin approval
  • Approved — Ready for disbursement
  • Active — Disbursed and being repaid
  • Defaulted — Overdue beyond grace period
  • Completed — Fully repaid
  • Rejected — Application declined by admin

Repayment Features

  • Flexible repayment via Mobile Money (MTN / Airtel)
  • Auto-generated schedules with installment breakdowns
  • Due date reminders via Email and SMS (5 days before due date)
  • Grace period before default marking
  • Partial repayments supported at any time
  • Payment history viewable in transaction history
3.5% base interest rate — reduced for high credit score tiers. YoCoin token holders get additional rate discounts.

Default Management

Automated default check runs on every request. Defaulted loans trigger notifications, credit score penalties, and admin alerts.

Mobile Money Integration
Seamless integration with Uganda's leading mobile money providers

Disbursement Flow

  • Admin approves loan → System creates disbursement request
  • Funds sent via MTN Mobile Money or Airtel Money API
  • Real-time status tracking: Pending → Processing → Completed / Failed
  • Automatic retry on failure (up to 3 attempts)
  • Transaction reference stored for reconciliation
  • User notified via SMS when funds are available
24-hour disbursement SLA. Most payments processed within 2-4 hours during business days.

Repayment Flow

  • User sends repayment via MTN / Airtel Mobile Money to YoCoin paybill
  • System detects incoming payment and auto-matches to user account
  • Payment applied to outstanding loan balance
  • Real-time confirmation via SMS and in-app notification
  • Partial repayments accepted and applied proportionally
  • Repayment history synced with blockchain
Mobile Money Providers: MTN Uganda & Airtel Uganda — covering 99% of Uganda's mobile money users.
